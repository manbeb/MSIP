### 2. Синтаксический разбор

**Выбор инструмента**: Библиотека `spaCy` с моделью `ru_core_news_md` для русского языка.

**Алгоритм извлечения пар (подлежащее, сказуемое)**:

```python
def extract_subject_predicate_pairs(doc):
    found_pairs = []
    for sent in doc.sents:
        # Поиск сказуемых (глаголы в позиции ROOT или conj)
        predicates = [token for token in sent 
                     if token.pos_ in ['VERB', 'AUX'] 
                     and token.dep_ in ["ROOT", "conj"]]
        
        # Для каждого сказуемого ищем подлежащее
        for pred in predicates:
            subject = None
            for child in pred.children:
                if child.dep_ in ["nsubj", "nsubj:pass"]:
                    subject = child
                    break
            # Сохраняем пару лемм в нижнем регистре
            if subject and subject.pos_ in ["NOUN", "PROPN", "PRON"]:
                found_pairs.append((subject.lemma_.lower(), pred.lemma_.lower()))
    return found_pairs
```

**Ключевые особенности**:
- Используются синтаксические зависимости spaCy: `nsubj` (активное подлежащее), `nsubj:pass` (пассивное подлежащее)
- Извлекаются только части речи: существительные, собственные имена, местоимения (для подлежащих) и глаголы/вспомогательные глаголы (для сказуемых)
- Применяется лемматизация для приведения слов к начальной форме

### 3. Многопоточная обработка

Для ускорения обработки 50 000 текстов использована встроенная многопоточность spaCy:

```python
docs_stream = nlp.pipe(texts, batch_size=50, n_process=9)
```

Время обработки: ~6 минут для 50 000 документов (~137 документов/сек).

### 5. Визуализация результатов

Реализованы три типа визуализаций:

1. **Горизонтальная столбчатая диаграмма ТОП-20 пар**

3. **Гистограмма распределения частот**
   - Показывает, что большинство пар встречаются редко (длинный хвост распределения)

## Анализ полученных результатов

### Лингвистические закономерности

1. **Жанровая специфика новостных текстов**:
   - Доминирование пар с юридической и криминальной лексикой: `дело — возбудить`, `обвинение — предъявить`, `суд — признать`
   - Частые упоминания инцидентов и происшествий: `инцидент — произойти`, `взрыв — произойти`, `человек — погибнуть`
   - Активное использование глаголов речи: `заявить`, `сообщать`, `отметить`

2. **Абстрактность подлежащих**:
   - Большинство частых подлежащих — абстрактные существительные (`дело`, `речь`, `решение`, `инцидент`)
   - Конкретные субъекты (`человек`, `глава`, `представитель`) встречаются реже

3. **Пассивные конструкции**:
   - Алгоритм корректно извлекает подлежащие из пассивных конструкций через зависимость `nsubj:pass`
   - Это важно для новостного стиля, где часто используется пассивный залог

4. **Лемматизация как ключевой этап**:
   - Без приведения к лемме одна и та же пара в разных формах считалась бы разной
   - Лемматизация позволила корректно агрегировать частоты

### Технические выводы

1. **Эффективность spaCy для русского языка**:
   - Модель `ru_core_news_md` показала хорошее качество синтаксического разбора
   - Многопоточная обработка через `nlp.pipe()` существенно ускорила анализ

2. **Масштабируемость подхода**:
   - Архитектура решения позволяет обрабатывать большие объёмы текстов
   - Использование `Counter` эффективно для подсчёта частот без загрузки всех данных в память одновременно

3. **Ограничения**:
   - Качество разбора зависит от качества входного текста (орфография, пунктуация)
   - Сложные предложения с эллипсисом или инверсией могут обрабатываться с ошибками
   - Не учитывается контекстуальная многозначность (омонимия)
