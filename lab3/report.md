# Отчет по Задаче 3: Фитнес-тренер (LangChain + LM Studio)

## 1. Архитектура решения
- **LLM**: `qwen/qwen3-4b-2507` через LM Studio (OpenAI-compatible endpoint).
- **Фреймворк**: LangChain v0.3+, FastAPI.
- **Тип агента**: `create_tool_calling_agent` с поддержкой JSON-вызова инструментов.

## 2. Системный промпт
Задает жесткую роль фитнес-тренера, требует использования инструментов перед ответом и содержит строгое правило: на нерелевантные запросы отвечать `"Команда неизвестна"`.

## 3. Подключенные инструменты (Skills)
1. `calculate_bmr` — расчет базового обмена веществ и суточной калорийности по формуле Миффлина-Сан Жеора.
2. `get_exercises_for_muscle` — возвращает готовую подборку упражнений с подходами/повторениями для 5 основных групп мышц.

## 4. Мини-сервис
REST API на FastAPI с эндпоинтом `POST /generate`. Принимает `message`, возвращает `response`. Готов к деплою или использованию в качестве бэкенда для фронтенда/мессенджеров.

## 5. Обработка неизвестных команд
Реализована на уровне системного промпта. При запросах вне тематики фитнеса модель возвращает строго: `"Команда неизвестна"`. Дополнительно агент защищен `handle_parsing_errors=True` для стабильной работы с локальными 4B-моделями.
